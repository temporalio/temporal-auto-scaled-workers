Queue consumer scaling simulator

Scaling rules

Scale up

A new consumer is created when all of the following are true:

  • There is at least one item in the queue.
  • No existing consumer has a free slot (all slots are busy).
  • The number of consumers is below Max consumers.
  • At least Scale up cooldown ms have elapsed since the last consumer was created.

Each new consumer is created with Slots per consumer slots.

Scale down

The most recently created consumer is removed when all of the following are true:

  • At least Scale down cooldown ms have elapsed since the last consumer was removed.
  • One of the two rate conditions holds:
    • n = 1: the dispatch rate over the last 10 s is 0.
    • n > 1: the current dispatch rate is below Scale-down threshold % of the peak rate ever observed with n − 1 consumers, meaning the extra consumer is not needed to keep up.

Peak rate tracking

Every tick the simulator records the dispatch rate (tasks started per second, measured over a 10 s rolling window) for each consumer count. The peak of all recorded samples is used by the scale-down rule to decide whether removing a consumer would keep the system able to handle the current load.
