Queue Consumer Scaling Simulator

Scaling rules

Scale-up

  1. On item arrival — if the queue is non-empty after dispatch, a consumer creation is attempted immediately.
  2. Scale check (every scale check interval):
    1. Dispatch rate non-zero and unchanged since the last check (within precision) → no action
    2. Queue depth > threshold → create consumer
    3. Queue non-empty and no consumer created within the last lifespan → create consumer
    4. Otherwise → no action

Scale-down

Each consumer stops after its lifespan expires.

Creation guards

Applied to every creation attempt before a consumer is launched:

  • Consumer count ≥ max consumers → blocked
  • Last creation within throttle window → blocked
  • Random roll < failure % → blocked
